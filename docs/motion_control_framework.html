速度与方向控制框架

在已有「串级平衡（速度外环 + 倾角内环）」之上，接入标准 ROS /cmd_vel， 实现前进/后退速度与偏航转向的框架层代码。 本文档说明架构、文件职责、话题接口与后续调参路线；具体增益需在站立稳定后再小步标定。

1. 控制分层总览

整体是三层解耦：平衡（必须）+ 速度（已有外环）+ 方向（新增偏航环）。

/cmd_vel (Twist) linear.x ──► MotionCommandAdapter ──► target_wheel_velocity [rad/s] angular.z ──► ──► target_yaw_rate [rad/s] │ ▼ ┌───────────────────────────────────────────────┐ │ on_timer @ control_rate (200 Hz) │ │ │ │ target_v ──► VelocityLoop (PI) ──► pitch_tgt │ │ ▲ │ │ wheel_v │ │ │ │ │ pitch_tgt ──► BalancePD (PD) ──► balance_tau │ │ ▲ │ │ pitch, pitch_rate │ │ │ │ target_w ──► YawLoop (PD) ──► turn_tau │ │ ▲ │ │ yaw_rate (IMU ωz) │ │ │ │ WheelMixer(balance, turn) ──► SafetyLimiter │ └───────────────────────────────────────────────┘ │ ▼ /wheel_effort_controller/commands [left τ, right τ]
框架原则：平衡环始终最高优先级；速度通过改 pitch_target 间接实现； 方向通过差动力矩实现，不直接改倾角目标，避免和平衡环抢 pitch。

2. 新增模块与文件

文件类职责
motion_command.py MotionCommandAdapter 解析 /cmd_vel，单位换算、限幅、超时归零
yaw_loop.py YawLoop target_yaw_rate − yaw_rate → turn_tau（差动力矩）
wheel_mixer.py WheelMixer 已有：left = balance − turn，right = balance + turn
balance_controller_node.py — 订阅 cmd_vel，每拍更新 target_v / target_w，串联各环
balance_types.py MotionCommand 内部运动指令数据结构；BalanceState 增加 yaw_rate

3. /cmd_vel 接口

字段单位含义内部转换
linear.xm/s前进为正，后退为负 target_wheel_velocity = linear.x / wheel_radius
angular.zrad/s逆时针为正（ROS 惯例） target_yaw_rate = angular.z（直接使用）
linear.y, linear.z, angular.x/y— 平面小车忽略

安全默认

  • cmd_vel_timeout_sec（默认 0.5 s）内无新消息 → 目标速度/角速度归零 → 车尝试站立；
  • max_linear_velocity、max_angular_velocity 对指令限幅；
  • motion_control_enabled: false 时忽略 /cmd_vel，退回 yaml 静态 target_velocity。
wheel_radius: 0.0325 m 与 URDF two_wheel_balance_65mm.urdf.xacro 一致。 例：linear.x = 0.1 m/s → 轮速期望 ≈ 0.1/0.0325 ≈ 3.08 rad/s。

4. 速度通道（复用串级外环）

外环 VelocityLoop 的 target_velocity 不再只来自 yaml，而是每拍由运动指令写入：

velocity_loop.config.target_velocity = motion_cmd.target_wheel_velocity

外环 PI 仍输出 pitch_target 给内环 PD——前进/后退的实现方式仍是「用倾角换轮速」，与站立平衡同一套物理机制。

给非零 linear.x 时，外环会要求更大的 pitch_target，更容易顶到 pitch_target_limit。运动调参前建议确认站立时 limit 有余量，或略提高 limit 并加强内环。

5. 方向通道（偏航环）

偏航 YawLoop 为角速度 PD（框架版）：

e_w = target_yaw_rate − yaw_rate
turn_tau = sign · ( Kp_yaw · e_w − Kd_yaw · d(yaw_rate)/dt )
turn_tau = clamp(turn_tau, ±max_turn_tau)
  • yaw_rate 来自 IMU angular_velocity.z；
  • turn_tau 为差动力矩：左轮减、右轮加（或相反，由 mix 公式决定）；
  • max_turn_tau 应明显小于 max_wheel_effort，避免转向抢光平衡力矩。
后续可扩展为：偏航角位置环（outer: yaw angle → target_yaw_rate，inner: 本 YawLoop）， 或接入 nav2 / 路径跟踪；当前框架先打通 cmd_vel 角速度通道。

6. WheelMixer 混合

balance_tau ──┐ ├──► left = balance_tau − turn_tau turn_tau ──┤ right = balance_tau + turn_tau │ 物理含义： 两轮同向分量 = 平衡；反向分量 = 转向

SafetyLimiter 对左右轮分别做力矩饱和与斜率限制。 若 |balance_tau| + |turn_tau| > max_wheel_effort，后饱和的一侧会被削顶—— 表现为「一边还能转、另一边已饱和」或转向变弱，属预期，需通过 max_turn_tau 与增益协调。

8. 如何测试

重新 build 一次（新增了 Python 模块）：

cd ~/ros2_projects/self_balance_car_class/two_wheel_self_balance_car
colcon build --packages-select balance_car_control --symlink-install
source install/setup.bash

8.1 启动 Gazebo + 平衡控制器

# 终端 1
ros2 launch balance_car_description gazebo.launch.py gui:=false

# 终端 2（等仿真起来后）
ros2 launch balance_car_control balance_controller.launch.py enabled:=true use_sim_time:=true

8.2 站立（零指令）

不发 /cmd_vel 或 timeout 后，target 自动为 0，行为与改框架前一致。

8.3 慢速前进

ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.08, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

8.4 原地转向

ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.3}}"

8.5 弧线（速度 + 方向）

ros2 topic pub --rate 10 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.1, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.2}}"
首次测试请从小速度、小角速度开始。转向反了改 yaw_output_sign； 前进方向反了检查 velocity_output_sign 与 linear.x 符号。

9. 调参顺序与后续扩展

9.1 推荐顺序

  1. 确认 motion_control_enabled: true 且无 cmd_vel 时仍能稳定站立；
  2. 小 linear.x（0.05~0.1 m/s）标速度通道，观察 debug[9]、[8] 是否跟踪；
  3. 小 angular.z（0.2~0.5 rad/s）标 kp_yaw，观察 debug[10]、[12]、[11]；
  4. 再试组合运动；若倒，先降 max_turn_tau 或 max_linear_velocity。

9.2 后续可扩展（未实现）

  • 偏航位置环：cmd_vel 积分 / 绝对 yaw 目标 → target_yaw_rate；
  • teleop launch：teleop_twist_keyboard + 本节点；
  • cmd_vel 平滑：S 曲线或低通，减轻阶跃指令对 pitch 环冲击；
  • 力矩预算分配：turn 与 balance 共享 max_wheel_effort 的智能缩放；
  • LQR / MPC：替换 balance + velocity 两环，保留 mixer 与 safety。
