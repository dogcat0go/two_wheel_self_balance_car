三环串级原理与位置环调参

在「速度环 + 倾角环」之上接入位置环，构成 位置 → 速度 → 倾角 三环串级，主要解决「停车时倒退一点、停不回原位」。 本文档说明三环原理、位置环实现与符号、以及位置环的逐步调参方法和现象对照。 对应代码：position_loop.py / velocity_loop.py / balance_pd.py。

1. 为什么要位置环

停车的物理过程（无法回避）：

前进中 → 要停 → 命令后仰刹车 → 车体后倾 → 轮子先“倒退一点”把质心追回竖直 → 速度归零 → 停下

纯速度环只保证 wheel_v → 0，不关心停在哪。 所以那一点刹车倒退量不会被补回来，车停在「倒退后的新位置」。

位置环的职责：记住一个目标位置 x_target，停车时把倒退的行程再开回来， 实现「停回原点」。这一步用 PID 即可，不需要 LQR。

2. 三环架构

v_cmd (来自 /cmd_vel 或静态 target_velocity) │ ▼ ┌──────────────────────────────────────────────────────────┐ │ 位置环 PositionLoop (最外, PD) │ │ x_target += v_cmd · dt （停车时 v_cmd=0 → 冻结） │ │ v_corr = kp_x·(x_target − x) + kd_x·(v_cmd − wheel_v) │ │ v_setpoint = v_cmd + v_corr │ └───────────────┬──────────────────────────────────────────┘ ▼ v_setpoint ┌──────────────────────────────────────────────────────────┐ │ 速度环 VelocityLoop (中间, PI) │ │ pitch_target = f(wheel_v − v_setpoint) │ └───────────────┬──────────────────────────────────────────┘ ▼ pitch_target ┌──────────────────────────────────────────────────────────┐ │ 倾角环 BalancePD (最内, PD) │ │ balance_tau = kp·(pitch − pitch_target) + kd·pitch_rate│ └───────────────┬──────────────────────────────────────────┘ ▼ balance_tau → Mixer → Safety → 轮子
环输入输出时间尺度
位置x 误差速度设定 v_setpoint最慢
速度速度误差目标倾角 pitch_target中
倾角pitch 误差力矩 τ最快
三环都在同一个 on_timer（200 Hz）里顺序执行：先算位置环 → 速度环 → 倾角环。 “外环更慢”靠的是增益小，不是更低的频率。带宽必须 位置 < 速度 < 倾角，串级才稳。

3. 位置环原理与公式

3.1 状态量

  • x = wheel_position：左右轮平均角位置 [rad]，来自 /joint_states 的 position；行程 ≈ x × wheel_radius。
  • x_target：目标位置 [rad]，由位置环内部维护。

3.2 控制律

x_target += v_cmd · dt （设定值积分，见下一节）
pos_error = x_target − x （并做 ±max_position_error 限幅 anti-windup）
v_corr = sign · ( kp_x · pos_error + kd_x · (v_cmd − wheel_v) )
v_corr = clamp(v_corr, ±max_velocity_correction)
v_setpoint = v_cmd + v_corr → 送入速度环

3.3 各项作用

项作用过大时
kp_x位置一偏就产生速度修正，把车开回 x_target停车点附近来回蹭、低频振荡
kd_x位置误差变化率阻尼（默认 0，阻尼交给速度环）对噪声敏感
max_velocity_correction限制位置环最多要求多大速度太大 → 回位猛、冲过头
max_position_error限制“记多远”，anti-windup太小 → 远距离回不来；太大 → 长时间追赶

3.4 为什么是“P 为主”

位置的微分就是速度，而速度已由速度环负责阻尼。所以位置环默认只用 P（kd_x=0）， 避免和速度环重复加阻尼、互相打架。需要更稳时再小量加 kd_x。

4. 设定值积分与符号（关键）

4.1 x_target 为什么要积分 v_cmd

工况v_cmdx_target 行为效果
站立 / 停车0冻结在停车点位置环把倒退量拉回 → 不留位移
前进 / 后退≠ 0以 v_cmd 速率移动位置环帮助跟踪行程，不和行驶打架

同一套逻辑兼顾「停回原位」与「正常行驶」，无需额外的状态切换。

4.2 符号约定

  • x、wheel_v、v_cmd 同一方向约定：正 = 前进（与 cmd_vel 的 linear.x 一致）。
  • 车倒退 → x 减小 → pos_error = x_target − x 为正 → v_corr 正（前进）→ 开回原点。
  • 若开启位置环后越跑越远 / 发散：把 position_output_sign 改为 -1.0。

4.3 使能 / 复位

以下情况 position_loop.reset(当前 x)，把 x_target 对齐当前轮位置，避免使能瞬间产生大误差甩车：

  • 安全停车（倒地 / IMU 超时）；
  • enabled=false 观察模式；
  • position_loop_enabled=false 时直通：v_setpoint = v_cmd（退回两环行为）。

6. 位置环调参步骤

前提：速度环已调稳（站立 |wheel_v| 小、停车基本不晃，只是会倒退一点）。 位置环是在“能停住”的基础上消除残余位移，不要用它救“站不住”。

6.1 起步配置

position_loop_enabled: true
kp_x: 0.4
kd_x: 0.0
max_velocity_correction: 3.0
max_position_error: 20.0
position_output_sign: 1.0

6.2 推荐步骤

  1. 先验证符号：站立 → 用手轻推车前进一点松手。位置环应让车自己开回原位附近。若反而加速跑开 → position_output_sign: -1.0。
  2. 加 kp_x 消倒退：从 0.4 起，每次 ×1.3。前进后停车，看停下后是否自动补回起步点（debug 里 x 回到 x_target）。
  3. 抑制来回蹭：若停车点附近反复前后蹭（低频），kp_x 偏大 → 回退一档；必要时加一点 kd_x（0.05~0.1）。
  4. 回位太猛：松手后冲过原点再荡回 → 降 max_velocity_correction（如 3.0→2.0）或降 kp_x。
  5. 行驶验证：发 cmd_vel 前进再停，确认行驶顺滑、停车不留明显位移、不与速度环打架。

6.3 验收标准

  • 前进后停车，最终停点与“开始减速点 + 命令行程”之差很小（残余位移小）；
  • 站立时 x 与 x_target 基本重合、不持续发散；
  • 无停车点附近的持续低频前后蹭。

7. 现象 → 改参对照

现象先看改（优先级）
开位置环后越跑越远 / 发散符号position_output_sign 翻 −1
停车仍明显倒退x 是否回 x_target加大 kp_x
停车点附近来回蹭（低频晃）kp_x 过大减 kp_x，可加一点 kd_x
松手回位冲过头再荡回回位速度太大降 max_velocity_correction
远距离推走后回不来误差被限幅加大 max_position_error
行驶时被“往回拽”、跑不动x_target 落后确认积分 v_cmd 生效；必要时降 kp_x
一开位置环就倒速度环本就不稳先关位置环调速度/倾角环

8. 测试命令

改 yaml 后重启控制节点即可（symlink-install 下不必重新 build；本次新增了 position_loop.py，首次需 build 一次）。

# 首次（新增模块）
colcon build --packages-select balance_car_control --symlink-install
source install/setup.bash

# 启动控制（非仿真时间）
ros2 launch balance_car_control balance_controller.launch.py enabled:=true use_sim_time:=false

前进一段再停（看是否停回行程末点、不倒退）

# 发 2 秒前进，然后 Ctrl-C 停止发布（超时后 v_cmd→0，位置环保持）
ros2 topic pub --rate 20 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.1, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

观察位置环

# 看 x(13) 与 x_target(14) 是否收敛，velocity_setpoint(15) 停车回位时短暂非零
ros2 topic echo /balance_controller/debug --field data

缓坡进阶（gentle_slope.world / balance_pid_test.world）

地形内嵌于 world 文件，由 scripts/generate_dual_slope_worlds.py 生成。步骤见根目录 README.md §5。

# 终端 1：平地起步上坡
colcon build --packages-select balance_car_description --symlink-install
source install/setup.bash
ros2 launch balance_car_description gazebo.launch.py pause:=true gui:=true \
  controller_type:=effort \
  world:=$(ros2 pkg prefix balance_car_description)/share/balance_car_description/worlds/gentle_slope.world \
  spawn_x:=-4.5 spawn_z:=0.034

# 终端 2：控制（与平地相同）
ros2 launch balance_car_control balance_controller.launch.py enabled:=true use_sim_time:=false

# 终端 3：unpause 后小速度上坡，到顶停发 cmd_vel 看是否溜坡
ros2 service call /unpause_physics std_srvs/srv/Empty {}
ros2 topic pub --rate 20 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.08, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"

岭台直接站立：spawn_x:=-0.5 spawn_z:=0.209（5°）；验收时看 pitch 稳态偏置、wheel_v 是否持续非零（溜坡）。

9. 局限与通往 LQR

  • 位置用 joint_states 轮角积分，无打滑/里程计漂移校正，长距离会累积误差；站立保位、短程足够。
  • 三环是分块近似：位置、速度、倾角分别 PID，环间耦合靠“外慢内快”保证，非全局最优。
  • LQR 把 [θ, θ̇, x, ẋ] 一起优化为 u = −Kx：天然包含位置项，可同时兼顾平衡与停位，是这套三环的统一升级。
  • 迁移时可复用现有 BalanceState（已含 pitch、pitch_rate、wheel_velocity、wheel_position）、safety 与 mixer，只替换控制律。
