1. 为什么用串级 PID

两轮自平衡车本质是倒立摆（inverted pendulum）装在驱动轮上： 要站稳，必须持续用轮子的力矩/加速度去“托住”倾角； 要停在原地，还必须把轮速压到期望值（通常 0）。

若只用单环倾角 PD（或 PD + 轮速阻尼 kv）：

  • 倾角可以暂时稳住，但质心偏移、IMU 零偏、仿真摩擦不对称会导致稳态前冲/后退；
  • 手工改 pitch_target 只能补偿一个方向，换模型/换电池位置又要重调；
  • kv 与“用倾角换加减速”的机制纠缠，增益一大容易和内环打架、缩短平衡时间。

串级 PID把问题拆成两层，各管一件事：

环路管什么输出实现文件
外环速度环 轮速是否等于期望（站立时 = 0） 目标倾角 pitch_target velocity_loop.py
内环倾角环 车体倾角是否跟上外环给的 pitch_target 单轮力矩 τ [N·m] balance_pd.py
执行 力矩限幅、斜率限制、倒地保护 左右轮指令 safety_limiter.py + ros2_control effort
工程结论：内环保证“不倒”；外环保证“不漂”。LQR 可在后续把两环合并为状态反馈，但串级 PID 更直观、便于分步调参，适合当前课程/仿真阶段。

2. 被控对象与物理直觉

2.1 简化模型

在竖直附近小角度线性化，可写成类似倒立摆 + 轮子的二阶/三阶系统（符号因坐标系而异）：

θ̈ ≈ f(θ, θ̇, τ_wheel)   （倾角动力学）
ω̇ ≈ g·θ / r − … + τ_wheel / (m·r²)   （轮速/位移与倾角耦合，示意）

精确系数取决于质心高度、总质量、轮半径、摩擦；仿真里还受 Gazebo 接触、力矩饱和、轮子是否打滑影响。 控制设计不依赖精确模型，但需理解两个耦合通道：

  • 倾角 θ：偏离竖直 → 重力矩增大 → 必须靠轮子力矩快速纠正，否则倒；
  • 轮速 ω：为维持 θ，轮子会持续转；质心不在轮轴正上方时，维持竖直所需的倾角 ≠ 0，车会匀速漂移，除非用倾角“换”加减速度把 ω 拉回 0。

2.2 “用倾角换加减速”

要让车从向前冲（ω > 0）慢下来，需要车体相对竖直方向形成适当的目标倾角， 使重力沿轮切向的分量产生减速趋势；外环 PI 的作用就是自动算出这个 pitch_target， 内环 PD 负责把车体稳定在该倾角附近。

非最小相位：你命令“更后仰”时，轮速不会立刻变为负，中间有相位滞后。 因此外环增益过大、积分过快会出现前冲 → 反向晃 → 慢收敛，这是串级系统的正常现象，靠减小 ki_v、适当加 kd_pitch 抑制。

2.3 自然平衡角 θnatural

当质心略偏前/IMU 有零偏时，维持机械平衡所需的倾角可能为 ±几度。 外环积分项会在稳态自动累积到该角度附近，相当于自动估计 θnatural， 因此 balance_controller.yaml 里的 pitch_target 偏置通常可设很小，仅作微调。

3. 总体控制架构

┌─────────────────────────────────────────┐ target_velocity ──►│ 外环 VelocityLoop (PI) │ wheel_v (滤波后) ──►│ 输入: e_v = ω − ω_ref │ │ 输出: pitch_target_cmd │ └──────────────────┬──────────────────────┘ │ pitch_target_bias ─┼──► pitch_target = bias + cmd ▼ ┌─────────────────────────────────────────┐ pitch, pitch_rate ─►│ 内环 BalancePD (PD) │ │ 输入: e_θ = θ − pitch_target │ │ 输出: balance_tau [N·m] │ └──────────────────┬──────────────────────┘ ▼ ┌─────────────────────────────────────────┐ │ WheelMixer → SafetyLimiter → 左右轮 τ │ └──────────────────┬──────────────────────┘ ▼ wheel_effort_controller (ros2_control)

数据来自：

  • /imu/data → AttitudeEstimator：pitch、pitch_rate、yaw；
  • /joint_states → 左右轮速平均 + 一阶低通 → wheel_velocity；
  • 定时器 control_rate（默认 200 Hz）→ on_timer 中串联两环。

4. 外环：速度 PI

实现：src/balance_car_control/src/velocity_loop.py，类型为 PI（无 D），输出为内环的 pitch_target。

4.1 误差定义（本车约定）

教科书常写 e = ωref − ω；本实现为：

e_v = ω − ω_ref = wheel_velocity − target_velocity

再经 velocity_output_sign（yaml 里可为 ±1）映射到 pitch_target。 符号必须通过实测确定：轮速向前冲时，外环应给出能“刹车”的倾角方向。 若漂移反而加剧，将 velocity_output_sign 改为相反号即可。

4.2 控制律

P_v = Kp_v · e_v
I_v 累加: ∫ e_v dt，I_term = Ki_v · ∫ e_v dt
pitch_target_raw = sign · (P_v + I_term)
pitch_target = clamp(pitch_target_raw, ±pitch_target_limit)

4.3 积分限幅（Anti-windup）

对 I_term 的贡献限制在 ±velocity_integral_limit， 并把积分状态回算为 integral = I_term / Ki_v，避免外环饱和后积分继续堆积（conditional clamp）。

当 |pitch_target| 顶到 pitch_target_limit 时，外环失去进一步“要求刹车倾角”的能力， 若此时 wheel_v 仍很大，说明需要更大限幅或更强内环跟踪，否则会倒。

4.4 外环各项物理意义

项作用过大时
Kp_v 轮速一偏离就立刻改目标倾角；抑制启动前冲 反应过猛、与内环相位叠加 → 高频抖
Ki_v 消除稳态漂移，自动找 θnatural 低频来回晃（hunting）、收敛慢
pitch_target_limit 保护内环，限制最大“刹车倾角” 太小 → 顶死仍刹不住；太大 → 内环跟不上易倒

4.5 复位条件

VelocityLoop.reset() 在以下情况调用，清空积分：

  • 安全停车（倒地、IMU 超时等）；
  • enabled=false 观察模式。

防止倒地期间积分饱和，重新 enable 时“甩头”。

# velocity_loop.py 核心（节选）
v_error = wheel_velocity - target_velocity
p_term = kp_v * v_error
_integral += v_error * dt
i_term = ki_v * _integral   # 并对 i_term 做 integral_limit
pitch_target = output_sign * (p_term + i_term)
pitch_target = clamp(pitch_target, ±pitch_target_limit)

5. 内环：倾角 PD

实现：src/balance_car_control/src/balance_pd.py。串级模式下 kv_wheel 应置 0，轮速完全交给外环。

5.1 控制律

e_θ = pitch − pitch_target
τ_balance = sign · ( Kp_pitch · e_θ + Kd_pitch · pitch_rate )

注意：本车实现为 +Kd·pitch_rate，与部分教材 cart-pole 公式的减号不同， 由 URDF/IMU 坐标系下 pitch_rate 与“倒下方向”的同号关系决定（见代码注释）。 调参时以加 Kd 是否更稳为准，不要机械套用教科书符号。

5.2 内环各项物理意义

参数单位作用
Kp_pitchN·m/rad倾角误差 → 力矩；决定“站得住”的基本刚度
Kd_pitchN·m/(rad/s)阻尼角速度；抑制摆动、帮助跟踪外环给的 pitch_target
pitch_targetrad节点里 = bias + 外环输出；外环开启时 bias 宜小

5.3 与力矩饱和、打滑

下游为 effort_controllers/JointGroupEffortController，max_wheel_effort 必须受地面摩擦限制。 若 τ 过大轮子空转，内环等效增益崩溃，串级两环同时失效。 经验：μ≈1、m≈0.76 kg、r≈32.5 mm 时单轮 slip 阈值约 0.12 N·m（yaml 中可能略放宽到 0.24 用于仿真裕度）。 

# balance_pd.py 核心（节选）
pitch_error = pitch - pitch_target
raw_effort = kp_pitch * pitch_error + kd_pitch * pitch_rate
return output_sign * raw_effort

6. 节点中的信号流与执行周期

实现：balance_controller_node.py 的 on_timer：

  1. 读取 BalanceState（IMU + 滤波轮速）；
  2. 安全检查（倾角过大、IMU 超时）→ 失败则 velocity_loop.reset() + 力矩置零；
  3. 若 enabled：pitch_cmd = velocity_loop.compute(ω, dt)；
  4. balance_pd.config.pitch_target = pitch_target_bias + pitch_cmd；
  5. τ = balance_pd.compute(state) → 混合左右轮 → 限幅 → 发布。

8. 调参方法与典型现象

8.1 推荐顺序

  1. 内环单独能站：外环 kp_v=ki_v=0 或 velocity_loop_enabled=false，调 kp_pitch、kd_pitch；
  2. 开外环，先 P 后 I：小 kp_v 抑制前冲；稳后再小步加 ki_v 消稳态漂移；
  3. 内环跟踪：若 pitch 长期跟不上 pitch_tgt，略增 kp_pitch 或 kd_pitch，并检查力矩是否顶饱和；
  4. 外环限幅：日志里 pitch_tgt 长时间钉在 ±limit 且 wheel_v 仍大 → 提高 limit 或加强内环/外环 P（谨慎）。

8.2 现象对照

现象可能原因建议
启动后向前冲一段 外环反应慢；积分从 0 爬升 略增 kp_v；勿同时猛加 ki_v
前冲后来回慢晃 ki_v 过大；环间相位滞后 减 ki_v；略增 kd_pitch
稳态 wheel_v 小常数 ≠ 0 积分不够或偏置未抵消 微增 ki_v 或微调 pitch_target bias
pitch_tgt 顶限幅仍飞车 外环“想刹”但内环/力矩跟不上 检查 max_wheel_effort、kp_pitch、limit
越调越倒、漂移加速 外环符号反了 翻转 velocity_output_sign

8.3 带宽直觉

内环带宽应明显高于外环（倾角响应比速度调节快一个数量级左右）， 这样外环改 pitch_target 时，内环能近似跟踪，串级分析才成立。 若内环太“软”（kp_pitch 小、力矩饱和），外环看到的就像“慢植物”，容易振荡。

9. 与单环 kv、与 LQR 的关系

9.1 串级 vs 单环 + kv

单环 PD + kv串级 PI(速度) + PD(倾角)
稳态漂移 靠手工 pitch_target 或 kv 勉强压 外环积分自动找平
速度设定 无显式 ω_ref target_velocity 可直接设前进/后退
调参 kp/kd/kv 耦合 先内后外，分工清晰

9.2 与 LQR 的衔接

LQR 对线性化模型 ẋ = Ax + Bu 求状态反馈 u = −Kx， 状态常取 [θ, θ̇, x, ẋ] 或含轮角速度的形式。 串级 PID 可看作对同一状态的分块、带限幅的近似解：

  • 外环 ≈ 对轮速/位移通道的积分反馈；
  • 内环 ≈ 对 θ、θ̇ 的 PD 反馈；
  • LQR 优势：多变量耦合一次优化、有稳定性理论；劣势：模型依赖、实现与调参门槛更高。

本仓库后续可在同一 BalanceState 上切换 Controller 策略，保留 safety 与 mixer 层不变。
