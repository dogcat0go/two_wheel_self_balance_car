两轮自平衡小车平衡环的物理原理

平衡车不是靠“反作用力直接把车身推正”这么简单。电机力矩、轮地摩擦、重力矩和支撑点移动同时作用。 角动量当然需要遵守守恒定律，但整车不是孤立系统：地面和重力持续给系统外力与外力矩。

一句话结论

平衡环的目标不是“给车身一个反作用力让它转回去”，而是通过轮子加速让轮轴移动， 让车轮接地点追到车身重心下方，从而让重力矩不再继续把车身摔倒。

电机做的事

电机在车身和轮子之间施加等大反向力矩。

内部力矩

地面做的事

轮子对地面施力，地面对轮子产生摩擦力，推动轮轴前后移动。

外力

重力做的事

重心不在接地点正上方时，重力对接地点产生翻倒力矩。

外力矩

你说的“反作用力让车身向反方向转动”是对的一部分：电机确实会给车身一个反向力矩。 但平衡车真正能长期站住，主要靠轮子和地面的摩擦把支撑点移动到合适位置。

结构示意图

下面用前后方向的侧视图表示。假设车身向前倾，重心在轮轴前方。 控制器会让轮子向前加速，使轮轴追向重心下方。

电机力矩和地面摩擦

电机安装在车身上，输出轴连着轮子。电机对轮子施加力矩时，车身会受到等大反向力矩。 这部分符合牛顿第三定律，也是你理解里的“反作用力”。

1. 电机对轮子施加力矩
轮子获得角加速度，试图相对地面滚动。
2. 轮子和地面产生静摩擦
如果不打滑，接地点相对地面近似静止，地面对轮子产生前后方向摩擦力。
3. 摩擦力推动轮轴平移
轮轴带着车身底部前后移动，支撑点位置发生变化。
4. 支撑点追到重心下方
重力对接地点的力臂变小，车身不再继续向同一方向摔倒。

对平衡来说，“车轮转起来”不是最终目的。真正有用的是轮地摩擦把轮轴位置改变了。

倒立摆视角

两轮平衡车可近似看作“倒立摆 + 可移动底座”。车身是倒立摆，轮轴是底座。 如果车身向前倾，控制器让底座向前加速；如果车身向后倾，控制器让底座向后加速。

现象 物理解释 控制动作
车身向前倾 重心落在接地点前方，重力产生继续前倒的力矩 轮子向前加速，让轮轴追上重心
车身向后倾 重心落在接地点后方，重力产生继续后倒的力矩 轮子向后加速，让轮轴追上重心
车身来回震荡 P 项过大或阻尼不足 增加 D 项，或者降低 P 项
车站住但一直跑 内环只管姿态，不管位置 后续加入轮速阻尼、速度外环或位置外环

角动量是否守恒

角动量守恒的前提是系统受到的外力矩为零。两轮平衡车在地面上运动时，这个前提不成立。 因为系统持续受到地面接触力、地面摩擦力、重力、法向力等外力。

如果只看“车身 + 轮子”

电机力矩是内部力矩，轮子和车身之间的角动量交换需要守恒。

局部守恒

如果加入“地面接触”

地面对轮子的摩擦力是外力，能改变整车线动量，也会对不同参考点产生外力矩。

外力参与

如果以接地点为转动参考

重力对接地点有力臂，会产生翻倒力矩；支撑点移动会改变这个力臂。

重力矩

所以不能简单说“角动量守恒，所以车轮转了车身就必然反向转同样多”。 在真实平衡车问题中，地面不断交换动量，重力不断施加翻倒力矩，电机只是让轮子和地面产生正确的相互作用。

平衡环在控制什么

平衡环本质上在控制车身相对竖直方向的角度和角速度。它通过输出轮速命令间接控制轮轴加速度。 

pitch_error = pitch - pitch_target
wheel_command = output_sign * (
    kp_pitch * pitch_error
    - kd_pitch * pitch_rate
)
publish([wheel_command, wheel_command])

其中 kp_pitch 负责“看到倾斜就追过去”，kd_pitch 负责“看到正在快速倒下就提前刹住”。第一版不用 yaw，也不建议用 I 项。

更精确的物理模型会输出轮子力矩，而你当前系统通过 velocity_controllers/JointGroupVelocityController 接收轮速命令。 因此第一版控制器输出的是“期望轮子角速度”，不是直接输出电机力矩。

代码变量对应物理量

代码变量 物理量 作用
pitch 车身前后倾角 判断重心偏在支撑点前方还是后方
pitch_rate 车身倾倒角速度 提供阻尼，避免越调越抖
kp_pitch 姿态刚度 倾角越大，轮轴追得越快
kd_pitch 姿态阻尼 倾倒越快，反向抑制越强
output_sign 方向约定修正 匹配 IMU pitch 正方向和轮速正方向
max_wheel_velocity 轮速饱和 避免轮子瞬间打满导致打滑或仿真震荡

调试时最重要的是先确认方向：车向前倒，轮子应该向前追；车向后倒，轮子应该向后追。 如果方向反了，先改 output_sign，不要急着改 PID 参数。
