两轮自平衡小车控制架构设计

当前项目已经具备 Gazebo 模型、IMU、关节状态反馈，以及 /wheel_velocity_controller/commands 左右轮速度入口。 下一步不是先追求复杂算法，而是先搭出清晰的闭环控制架构：读取姿态，估计车身倾角，计算左右轮角速度，让轮子把车身“接住”。

目标和已知条件

现阶段的目标是让小车在仿真中从轻微倾斜状态恢复到接近竖直，并能在扰动后继续保持平衡。 已经确认的能力是：向 /wheel_velocity_controller/commands 发布 std_msgs/msg/Float64MultiArray，可以让左右轮转动。

已有输入

IMU 和关节状态已经可作为反馈源。

/imu/data /imu_sensor_broadcaster/imu /joint_states

已有输出

左右轮角速度命令已经能进入 Gazebo 物理仿真。

/wheel_velocity_controller/commands Float64MultiArray

关键约束

URDF 中轮子速度限制为 35 rad/s，控制器更新频率为 200 Hz，IMU 更新频率为 250 Hz。

速度限幅 实时性

坐标约定已经在模型中定义为 x 向前、y 向左、z 向上。 小车前后倒下时，主要姿态量是绕 y 轴的 pitch；控制轮子时，也是通过左右轮同向转动让轮轴追上车身重心。

平衡原理

两轮自平衡车可以近似看成“倒立摆 + 可移动轮轴”。当车身向前倾时，轮子必须向前加速， 让轮轴移动到重心下方；当车身向后倾时，轮子必须向后加速。控制的本质不是“让轮子停住”， 而是通过轮子运动不断调整底部支撑点。

1. IMU 测出车身倾角和角速度
从 IMU orientation 得到 pitch，从 angular_velocity.y 得到 pitch rate。
2. 控制器判断车身正在往哪边倒
如果 pitch 偏离竖直，控制器计算需要的轮子速度方向和大小。
3. 左右轮同向转动
平衡时左右轮通常给同一个速度值，例如 [w, w]，让车沿前后方向移动。
4. 物理反馈回到 IMU
Gazebo 更新车身姿态，IMU 再把新姿态反馈给控制器，形成闭环。

小车要保持平衡，轮子一定会来回移动。真正稳定后，车身 pitch 接近 0， 但轮子位置不一定固定在原地。后续如果要“原地平衡”，还需要增加位置或速度外环。

IMU 调试 world

两轮平衡车在没有控制器时是天然不稳定的倒立摆，加载进 Gazebo 后会很快倒下。 如果直接在自由跌倒过程中验证 IMU，orientation、angular velocity 和接触碰撞都会混在一起， 很难判断 pitch 计算是否正确。因此新增一个专用 world： src/balance_car_description/worlds/imu_debug.world。

控制节点内部结构

第一版可以只有一个 ROS2 节点，但节点内部要分层。这样既方便调试，又避免一开始就拆成太多 ROS 节点造成复杂度。

timer_callback():
  state = estimator.update(latest_imu, latest_joint_states)
  raw_balance_w = controller.compute(state)

ROS 接口设计

第一版只需要订阅已有反馈话题，并发布现有轮速命令。调试话题建议从一开始保留， 因为平衡控制主要靠观察 pitch、pitch_rate 和输出速度来调参。

方向 话题 类型 用途
订阅 /imu/data sensor_msgs/msg/Imu 第一优先级 IMU 输入。也可以切换到 /imu_sensor_broadcaster/imu。
订阅 /joint_states sensor_msgs/msg/JointState 读取左右轮速度和位置，用于速度外环、调试和状态判断。
发布 /wheel_velocity_controller/commands std_msgs/msg/Float64MultiArray 发布左右轮角速度，顺序为 [left_wheel_joint, right_wheel_joint]。
发布 /balance_controller/debug 建议先用 std_msgs/msg/Float64MultiArray 输出 pitch、pitch_rate、控制量、限幅后的命令，便于画图调参。
可选订阅 /cmd_vel geometry_msgs/msg/Twist 未来加入遥控目标速度和转向，不作为第一阶段必要功能。

当前输出单位是轮子关节角速度 rad/s，不是底盘线速度。 车轮半径为 0.0325 m，所以可用 wheel_w = body_v / 0.0325 在底盘速度和轮子角速度之间换算。

控制算法分层

建议按三个阶段推进：先能扶住，再能少跑，再能听从速度命令。每个阶段保留同一套代码接口， 只是逐步打开更多控制项。

阶段 1：只做姿态 PD

balance_w = kp_pitch * pitch_error
          + kd_pitch * pitch_rate

这里 pitch_error = pitch - pitch_target，pitch_target 初始为 0。 实际符号需要通过小幅测试确定：如果车向前倒，轮子却向后跑，就把整体输出符号反过来。

阶段 2：加入轮速阻尼

balance_w = kp_pitch * pitch_error
          + kd_pitch * pitch_rate
          + kv_wheel * average_wheel_velocity

轮速项用来减少小车越跑越远的问题，相当于给底盘速度加阻尼。

阶段 3：加入位置或速度外环

pitch_target = velocity_pid.compute(target_velocity - measured_velocity)
balance_w = balance_pid.compute(pitch_target - pitch, pitch_rate)

这个阶段才适合接入 /cmd_vel。外环不要直接控制轮子，而是给内环一个很小的目标倾角， 让车身通过“略微前倾/后倾”实现加速和减速。

如果以后想走更系统的路线，可以把状态定义成 [pitch, pitch_rate, wheel_position, wheel_velocity]，再用 LQR 求控制增益。 但第一版用 PD/PID 更适合快速理解方向、符号、限幅和传感器反馈。

IMU 与 PID 调试代码

这一节给出第一版可以直接写入 balance_car_control 包的代码骨架。 推荐先运行 imu_debug_node.py，确认 pitch 符号和 angular_velocity.y 正确； 再运行 pid_debug_node.py，用很小的速度限幅确认轮子输出方向。

imu_debug_node.py

这个节点只做 IMU 姿态解算，不发轮子命令。它把 quaternion 转成 roll、pitch、yaw， 同时发布 /balance_controller/imu_debug，数组顺序为 [roll, pitch, yaw, angular_velocity.x, angular_velocity.y, angular_velocity.z]。 

import math

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Imu
from std_msgs.msg import Float64MultiArray


def quaternion_to_euler(x, y, z, w):
    sinr_cosp = 2.0 * (w * x + y * z)
    cosr_cosp = 1.0 - 2.0 * (x * x + y * y)
    roll = math.atan2(sinr_cosp, cosr_cosp)

    sinp = 2.0 * (w * y - z * x)
    if abs(sinp) >= 1.0:
        pitch = math.copysign(math.pi / 2.0, sinp)
    else:
        pitch = math.asin(sinp)

    siny_cosp = 2.0 * (w * z + x * y)
    cosy_cosp = 1.0 - 2.0 * (y * y + z * z)
    yaw = math.atan2(siny_cosp, cosy_cosp)

    return roll, pitch, yaw


class ImuDebugNode(Node):
    def __init__(self):
        super().__init__("imu_debug_node")
        self.declare_parameter("imu_topic", "/imu/data")
        self.declare_parameter("print_degrees", True)
        self.declare_parameter("log_period_sec", 0.25)

        imu_topic = self.get_parameter("imu_topic").value
        self.print_degrees = self.get_parameter("print_degrees").value
        self.log_period_sec = float(self.get_parameter("log_period_sec").value)

        self.last_log_time = self.get_clock().now()
        self.debug_pub = self.create_publisher(
            Float64MultiArray,
            "/balance_controller/imu_debug",
            10,
        )
        self.create_subscription(Imu, imu_topic, self.on_imu, 10)

        self.get_logger().info(f"Listening IMU topic: {imu_topic}")

    def on_imu(self, msg):
        q = msg.orientation
        roll, pitch, yaw = quaternion_to_euler(q.x, q.y, q.z, q.w)

        av = msg.angular_velocity
        self.debug_pub.publish(Float64MultiArray(data=[
            roll,
            pitch,
            yaw,
            av.x,
            av.y,
            av.z,
        ]))

        now = self.get_clock().now()
        elapsed = (now - self.last_log_time).nanoseconds / 1e9
        if elapsed < self.log_period_sec:
            return

        self.last_log_time = now
        if self.print_degrees:
            self.get_logger().info(
                "roll={:.2f} deg, pitch={:.2f} deg, yaw={:.2f} deg, "
                "gyro_y={:.4f} rad/s".format(
                    math.degrees(roll),
                    math.degrees(pitch),
                    math.degrees(yaw),
                    av.y,
                )
            )
        else:
            self.get_logger().info(
                "roll={:.4f}, pitch={:.4f}, yaw={:.4f}, gyro_y={:.4f}".format(
                    roll,
                    pitch,
                    yaw,
                    av.y,
                )
            )


def main():
    rclpy.init()
    node = ImuDebugNode()
    try:
        rclpy.spin(node)
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == "__main__":
    main()

pid.py

PID 工具类需要包含积分限幅、输出限幅和 reset。第一阶段可以只用 P 和 D， 但把 I 项接口保留，后续做速度外环或位置外环时会用到。 

from dataclasses import dataclass
from typing import Optional


@dataclass
class PIDConfig:
    kp: float = 0.0
    ki: float = 0.0
    kd: float = 0.0
    integral_limit: Optional[float] = None
    output_limit: Optional[float] = None


class PID:
    def __init__(self, config: PIDConfig):
        self.config = config
        self.integral = 0.0
        self.previous_error = None

    def reset(self):
        self.integral = 0.0
        self.previous_error = None

    def compute(self, error, dt, measured_rate=None):
        if dt <= 0.0:
            return 0.0

        p_term = self.config.kp * error

        self.integral += error * dt
        if self.config.integral_limit is not None:
            limit = abs(self.config.integral_limit)
            self.integral = max(-limit, min(limit, self.integral))
        i_term = self.config.ki * self.integral

        if measured_rate is not None:
            d_error = -measured_rate
        elif self.previous_error is None:
            d_error = 0.0
        else:
            d_error = (error - self.previous_error) / dt
        d_term = self.config.kd * d_error

        self.previous_error = error
        output = p_term + i_term + d_term

        if self.config.output_limit is not None:
            limit = abs(self.config.output_limit)
            output = max(-limit, min(limit, output))

        return output

pid_debug_node.py

这个节点是最小闭环调试版本：订阅 IMU，计算 pitch，使用 PD 输出一个左右轮同向速度。 为了安全，默认 enabled 为 false，确认 IMU 正确后再通过参数打开。 

import math

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Imu
from std_msgs.msg import Float64MultiArray

from balance_car_control.pid import PID, PIDConfig
from balance_car_control.imu_debug_node import quaternion_to_euler


class PidDebugNode(Node):
    def __init__(self):
        super().__init__("pid_debug_node")

        self.declare_parameter("imu_topic", "/imu/data")
        self.declare_parameter("command_topic", "/wheel_velocity_controller/commands")
        self.declare_parameter("debug_topic", "/balance_controller/pid_debug")
        self.declare_parameter("control_rate", 100.0)
        self.declare_parameter("enabled", False)
        self.declare_parameter("output_sign", 1.0)
        self.declare_parameter("pitch_target", 0.0)
        self.declare_parameter("kp_pitch", 2.0)
        self.declare_parameter("kd_pitch", 0.15)
        self.declare_parameter("max_wheel_velocity", 3.0)
        self.declare_parameter("fall_angle_rad", 0.61)
        self.declare_parameter("imu_timeout_sec", 0.1)

        self.enabled = bool(self.get_parameter("enabled").value)
        self.output_sign = float(self.get_parameter("output_sign").value)
        self.pitch_target = float(self.get_parameter("pitch_target").value)
        self.max_wheel_velocity = float(self.get_parameter("max_wheel_velocity").value)
        self.fall_angle_rad = float(self.get_parameter("fall_angle_rad").value)
        self.imu_timeout_sec = float(self.get_parameter("imu_timeout_sec").value)

        kp = float(self.get_parameter("kp_pitch").value)
        kd = float(self.get_parameter("kd_pitch").value)
        self.pid = PID(PIDConfig(
            kp=kp,
            ki=0.0,
            kd=kd,
            output_limit=self.max_wheel_velocity,
        ))

        imu_topic = self.get_parameter("imu_topic").value
        command_topic = self.get_parameter("command_topic").value
        debug_topic = self.get_parameter("debug_topic").value

        self.latest_pitch = None
        self.latest_pitch_rate = None
        self.last_imu_time = None
        self.last_control_time = self.get_clock().now()

        self.command_pub = self.create_publisher(Float64MultiArray, command_topic, 10)
        self.debug_pub = self.create_publisher(Float64MultiArray, debug_topic, 10)
        self.create_subscription(Imu, imu_topic, self.on_imu, 10)

        control_rate = float(self.get_parameter("control_rate").value)
        self.create_timer(1.0 / control_rate, self.on_timer)

        self.get_logger().warn(
            "PID debug node started. enabled={} max_wheel_velocity={:.2f} rad/s".format(
                self.enabled,
                self.max_wheel_velocity,
            )
        )

    def on_imu(self, msg):
        q = msg.orientation
        _, pitch, _ = quaternion_to_euler(q.x, q.y, q.z, q.w)
        self.latest_pitch = pitch
        self.latest_pitch_rate = msg.angular_velocity.y
        self.last_imu_time = self.get_clock().now()

    def publish_wheel_command(self, wheel_velocity):
        self.command_pub.publish(Float64MultiArray(data=[
            wheel_velocity,
            wheel_velocity,
        ]))

    def stop_wheels(self):
        self.publish_wheel_command(0.0)
        self.pid.reset()

    def on_timer(self):
        now = self.get_clock().now()
        dt = (now - self.last_control_time).nanoseconds / 1e9
        self.last_control_time = now

        if self.latest_pitch is None or self.last_imu_time is None:
            self.stop_wheels()
            return

        imu_age = (now - self.last_imu_time).nanoseconds / 1e9
        if imu_age > self.imu_timeout_sec:
            self.get_logger().warn("IMU timeout, stopping wheels", throttle_duration_sec=1.0)
            self.stop_wheels()
            return

        pitch = self.latest_pitch
        pitch_rate = self.latest_pitch_rate or 0.0
        pitch_error = pitch - self.pitch_target

        if abs(pitch) > self.fall_angle_rad:
            self.get_logger().warn(
                "Fall angle reached: pitch={:.2f} deg".format(math.degrees(pitch)),
                throttle_duration_sec=1.0,
            )
            self.stop_wheels()
            return

        raw_output = self.pid.compute(
            error=pitch_error,
            dt=dt,
            measured_rate=pitch_rate,
        )
        wheel_velocity = self.output_sign * raw_output
        wheel_velocity = max(
            -self.max_wheel_velocity,
            min(self.max_wheel_velocity, wheel_velocity),
        )

        if self.enabled:
            self.publish_wheel_command(wheel_velocity)
        else:
            self.stop_wheels()

        self.debug_pub.publish(Float64MultiArray(data=[
            pitch,
            pitch_rate,
            pitch_error,
            raw_output,
            wheel_velocity,
            1.0 if self.enabled else 0.0,
        ]))


def main():
    rclpy.init()
    node = PidDebugNode()
    try:
        rclpy.spin(node)
    finally:
        node.stop_wheels()
        node.destroy_node()
        rclpy.shutdown()


if __name__ == "__main__":
    main()

安全和调参

平衡控制很容易因为参数、符号或传感器延迟导致轮子瞬间打满，所以安全层必须从第一版就有。

速度限幅

命令限制在 URDF 的 [-35, 35] rad/s 内，第一轮调试建议先限制到 [-8, 8]。

max_wheel_velocity

倾角保护

如果 |pitch| 超过例如 35 deg，认为已经跌倒，停止输出或进入 recovery 模式。

fall_angle

传感器超时

如果 IMU 超过 0.1 s 没有更新，立即发布 [0, 0] 并报警。

imu_timeout

输出斜率限制

限制相邻周期速度变化，避免仿真中轮地接触瞬间打滑或数值震荡。

accel_limit

调参顺序建议固定：先确认 pitch 符号，再确认轮子命令方向，然后只调很小的 kp_pitch， 最后慢慢加入 kd_pitch。不要一开始就同时调多个参数。

实施路线

  1. 先用 imu_debug.world 加载临时支撑，避免小车一出生就倒下。
  2. 新增 balance_car_control 包，实现 imu_debug_node.py，确认节点时钟、IMU 话题和 pitch 符号正常。
  3. 使用 rqt_plot 观察 /balance_controller/imu_debug，确认竖直时 pitch 接近 0，前后倾倒方向符合预期。
  4. 加入 pid.py 和 pid_debug_node.py，默认 enabled:=false，先只看控制器计算出的输出。
  5. 在支撑 world 中打开 enabled:=true，把 max_wheel_velocity 限制到 3 rad/s 左右，确认轮子追车体方向。
  6. 如果车向前倒时轮子反向跑，只改 output_sign，不要同时改 PID 参数。
  7. 切回 empty.world，逐步增大 kp_pitch，再加入 kd_pitch 抑制震荡，最后加入轮速阻尼。
  8. 稳定后再接入 /cmd_vel，通过外环把目标速度变成目标倾角。

这套架构的核心是保持接口稳定：上游读取 IMU 和关节状态，下游发布左右轮角速度。 控制算法可以从 PD 升级到 PID、LQR，甚至 ros2_control 自定义 controller，但不需要推翻整体数据流。
