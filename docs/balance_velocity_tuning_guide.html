0. 调参前检查（比增益更重要）

增益再合理，下面任一条不满足，调参都会像「随机」：

检查项命令 / 位置期望
控制环实际频率 ros2 topic hz /wheel_effort_controller/commands 接近 control_rate（200 Hz）；若只有 ~20 Hz，先修仿真 RTF / /clock
仿真时间 yaml use_sim_time: true + launch 与 Gazebo 一致；/clock 不应长期 ~10 Hz
执行模式 effort 控制器 力矩模式；速度模式在 Gazebo 里常站不住
力矩物理上限 max_wheel_effort ≤ 地面摩擦 slip 阈值（本车约 0.12 N·m）；过大 → 轮子空转，PD 失效
控制器已 enable enabled:=true 日志 enabled=True published=True
打滑判据：PlotJuggler 里 balance_tau 很大但 pitch 仍快速发散、 wheel_v 异常飙升 → 多半不是 kp 太小，而是力矩已超出摩擦圆，先降 max_wheel_effort。

1. 直立环（内环）原理 BalancePD

1.1 控制对象

倒立摆：车体倾角 θ（IMU pitch）偏离目标时，通过轮地力矩把 θ 拉回。内环输出单轮纵向力矩 τ [N·m]。

1.2 控制律（本车实现）

e_θ = pitch − pitch_target
τ_balance = output_sign · ( Kp_pitch · e_θ + Kd_pitch · pitch_rate )
  • pitch_target = pitch_target_bias（yaml）+ 外环输出；仅调直立环时可固定为常数（如 0）。
  • pitch_rate = IMU angular_velocity.y，是倾角变化率，不是轮速。
  • 本车用 +Kd·pitch_rate（非教科书减号），由 URDF/IMU 坐标约定决定；以「加 Kd 是否减振」为准。
  • 串级模式下 kv_wheel = 0，轮速交给外环，避免两环抢同一职责。

1.3 等效二阶系统直觉

小角度下可把直立环近似为二阶系统：

自然频率 ω_n ∝ √(Kp · g / L)  （示意，L 为等效摆长）
阻尼比 ζ ∝ Kd / (2√Kp)  （示意）
  • Kp 大 → 响应快、刚度大 → 太小站不住，太大高频抖、易饱和。
  • Kd 大 → 抑制 θ̇、减振 → 太大对噪声敏感、输出毛刺多。

1.4 物理硬约束

单轮最大可用力矩（不打滑）：

τ_max ≈ μ · m · g / 2 · r ≈ 1.0 × 0.756 × 9.81 / 2 × 0.0325 ≈ 0.12 N·m

内环需要的 τ 超过此值时，轮子空转，pitch 反馈与力矩指令失去因果关系。

2. 直立环调参方法

目标：外环关闭或 kp_v=ki_v=0、pitch_target 固定时， 小车能短暂维持 |pitch| < 10°~15° 不立刻倒（允许漂移，不要求停住）。

2.1 隔离设置

# balance_controller.yaml 片段 —— 只调直立环
velocity_loop_enabled: false   # 或 kp_v: 0.0  ki_v: 0.0
pitch_target: 0.0              # 先不要手调偏置
kv_wheel: 0.0
kp_pitch: 0.5                  # 保守起点
kd_pitch: 0.02

2.2 推荐步骤

  1. 只加 Kp：从 0.3~0.5 起，每次 ×1.2~1.5。观察能站多久、是否高频抖。出现持续振荡 → 回退 20%。
  2. 再加 Kd：从 Kp 的 3%~8% 量级起（如 Kp=1.0 → Kd=0.03~0.08）。减轻左右/前后摆；若输出噪声大 → 略降 Kd 或检查 IMU。
  3. 检查饱和：倒之前 |balance_tau| 是否长期顶在 max_wheel_effort。是 → 先降 max 或降 Kp，不要继续加 Kp。
  4. 记录工作点：记下「能站最久」的 Kp/Kd，作为外环开启后的内环基准，外环阶段尽量不再大改 Kp。

2.3 内环参数旋钮表

参数单位加大后的效果过大时
kp_pitchN·m/rad倾角恢复更快高频振荡、力矩饱和、倒得更快
kd_pitchN·m/(rad/s)阻尼增大、摆动减少对 IMU 噪声敏感、τ 毛刺
pitch_target biasrad固定倾角偏置（trim）串级下宜小；主要靠外环积分找平
max_wheel_effortN·m允许更大力矩超过摩擦 → 打滑，闭环失效
max_effort_slewN·m/s—太小 → 响应迟钝；太大 → 阶跃冲击

2.4 内环合格标准（进入外环前）

  • 无漂移目标下，|pitch| 在 5°~15° 内能维持数秒；
  • balance_tau 多数时间未顶饱和；
  • 无明显持续等幅振荡（说明 Kp/Kd 比例大致合理）。

3. 速度环（外环）原理 VelocityLoop

3.1 为什么需要外环

仅直立环能「扶住角度」，但质心偏移 / IMU 零偏会使车匀速漂移（wheel_v → 常数 ≠ 0）。 外环通过改变 pitch_target，用「倾角换加减速」，把轮速拉回 target_velocity。

wheel_v ──► e_v = ω − ω_ref ──► PI ──► pitch_target ──► 内环 PD ──► τ ▲ │ └──────── 物理：倾角→轮速 ──────┘

3.2 控制律（本车实现）

e_v = wheel_velocity − target_velocity
pitch_target = sign · ( Kp_v · e_v + Ki_v · ∫e_v dt )
pitch_target = clamp(pitch_target, ±pitch_target_limit)
∫ 项另做 velocity_integral_limit anti-windup

3.3 各分项物理意义

项作用时间尺度
Kp_v · e_v轮速一偏就立刻改目标倾角；抑制启动前冲快（比例）
Ki_v · ∫e_v消除稳态漂移；自动找到 θ_natural慢（积分）
pitch_target_limit外环最多要求的倾角；保护内环安全护栏

3.4 符号 velocity_output_sign

平衡车是非最小相位系统，外环符号必须实测。你当前 yaml 为 -1.0。 若开外环后漂移加剧，在 +1 与 −1 之间切换。判据：wheel_v 为正且前冲时， pitch_target 应朝能刹车的方向变化，且 |wheel_v| 随时间减小。

3.5 带宽关系

外环应比内环慢：内环几十 ms 级跟踪 pitch_target；外环几百 ms~数 s 级调节 wheel_v。 若 Kp_v/Ki_v 过大，会出现「前冲 → 来回晃 → 慢收敛」的 hunting。

4. 速度环调参方法

目标：站立时 |wheel_v| 稳态 < 0.2 rad/s，|pitch − pitch_target| < 1°， 启动前冲距离短，无明显低频来回晃。

4.1 开启外环

velocity_loop_enabled: true
target_velocity: 0.0
kp_v: 0.005          # 从小开始
ki_v: 0.0            # 先关积分，只调 P
pitch_target_limit: 0.12
velocity_output_sign: -1.0   # 按你车实测

4.2 推荐步骤

  1. 先 Kp_v，Ki_v=0：0.003 → 0.008 → 0.015… 观察前冲是否变短、pitch_tgt 是否随 wheel_v 合理变化。抖 → 回退。
  2. 确认符号：前冲时 pitch_tgt 与 wheel_v 变化应「对抗漂移」。不对 → 翻转 velocity_output_sign。
  3. 再加 Ki_v：从 0.0005 起。消除稳态 wheel_v 残差；出现慢晃 → 减 Ki_v。
  4. 查限幅：若 pitch_tgt 长期钉在 ±6.88°（0.12 rad）且 wheel_v 仍大 → 略提 pitch_target_limit 或加 Kp_v，并确认内环跟得上（|pitch−pitch_tgt| 不大）。
  5. 微调 bias：外环积分仍有余量时，pitch_target bias 仅作 ±0.01 rad 级 trim，不要替代 Ki_v。

4.3 外环参数旋钮表

参数典型起点加大后过大时
kp_v0.005~0.017前冲更短、速度误差响应更快与内环耦合振荡
ki_v0.0005~0.002稳态 wheel_v→0，自动找平低频来回晃、收敛慢
pitch_target_limit0.10~0.15 rad允许更大「刹车倾角」内环跟不上 → 倒
wheel_velocity_lpf_alpha0.3~0.5大 → 轮速反馈更跟手小 → 滞后大、外环反应慢

4.4 你曾遇到的两类现象（对照）

现象日志特征优先动作
外环顶死仍飞车 pitch_tgt=6.88° 不变，wheel_v 持续增大 提 limit 或 Kp_v；加强内环；检查 sign
能站但前冲后慢晃 pitch_tgt 小幅正负切换，wheel_v 缓慢收敛 减 ki_v；略增 kd_pitch
稳态 wheel_v ≈ −0.3 rad/s 小常数 pitch_tgt 稳定，τ 很小 微增 ki_v 或微调 bias

5. 整体顺序与隔离原则

阶段 A：只调内环（外环关 / kp_v=ki_v=0） ↓ 能短暂站立 阶段 B：外环只开 Kp_v（ki_v=0） ↓ 前冲缩短、符号正确 阶段 C：小步加 Ki_v ↓ 稳态 wheel_v→0 阶段 D：（可选）cmd_vel 速度 / 偏航环
黄金规则：每次只改一个参数；改完跑同一段 rosbag（spawn → 平衡 → 漂移/倒）。 内环定型后，外环阶段避免同时大改 kp_pitch 和 ki_v。

5.1 修改 yaml 后如何生效

参数在 yaml 中：重启节点即可（symlink-install 下不必 colcon build）。

ros2 launch balance_car_control balance_controller.launch.py enabled:=true use_sim_time:=true

6.3 推荐曲线组合

目的画哪些看什么
内环是否跟得上 pitch, pitch_target 两条是否贴近；lag 大 → 加 kp_pitch 或检查 τ 饱和
内环是否过振 pitch, balance_tau 等幅振荡 → 降 kp 或加 kd
外环是否在工作 wheel_v, pitch_target wheel_v 偏时 pitch_tgt 应反向「拉」
外环是否顶死 pitch_target, wheel_v pitch_tgt 平台 + wheel_v 仍升 → limit/Kp_v
控制频率 debug 时间戳差分 应 ≈ 5 ms；若 ~50 ms 先修 clock
